## 🎯 Contexto

A tarefa **10 do board Odoo #1723** ("Demais APIs consumidas") pede cada API extra **em ordem separada**, e cita como exemplos a **API de script** (coberta na Story 49.7 — ponte CAPI `heziom-api`) e a **API de MultiCD** (multi centro de distribuição). O JG pediu explicitamente para incluir a MultiCD.

**Achado (código, 15/07):** o HeziomOS **NÃO consome hoje a API de Multi-CD da Tray**. O estoque é tratado como um campo único (`stock`) no espelho e na projeção (`_shared/tray-products.ts:47,77,90,110`) — não há chamadas a endpoints de centro de distribuição / estoque por CD, nem `PUT` de estoque por-CD. (A escrita de estoque simples `PUT /products/:id {stock}` é do Epic 38; MultiCD é outro recurso.)

**Portanto, a resposta à Tray depende de uma decisão de negócio (pendente do JG):**
- **(A) A Heziom NÃO opera múltiplos CDs** → declarar formalmente à Tray, na tarefa 10, que a **API de MultiCD não é consumida** (com justificativa: operação de CD único). Não há código a implementar.
- **(B) A Heziom opera / vai operar múltiplos CDs** → implementar o consumo da API MultiCD da Tray (listar CDs, ler/escrever estoque por CD), com store-agnostic + limitador de 180/min (49.4), e gerar a evidência (código + console). Isso vira Feature com Security Gate e provavelmente se conecta ao módulo de Estoque (Epic 31) e ao Move Gourmet (multi-CD já existe lá como referência).
