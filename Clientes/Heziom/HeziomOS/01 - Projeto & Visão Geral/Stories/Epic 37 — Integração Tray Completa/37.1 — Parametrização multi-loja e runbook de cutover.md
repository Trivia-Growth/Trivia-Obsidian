# 37.1 — Parametrização multi-loja e runbook de cutover

## Contexto
> A virada teste→produção só pode ser troca de config. Hoje há valores de loja espalhados: `TRAY_API_HOST` aponta para a loja de teste, `seller_id`/`app_code` de exemplo (1501119/8395), `event_source_url` hardcoded na `heziom-api`, e o `api_host` varia por loja mas nem sempre é lido do `/auth`. Esta story torna toda a integração **store-agnostic** e produz o runbook de cutover.

## Spec de Referência
- [[Tray — Auditoria de Capacidades vs Produção]] (seção Bloqueadores + Checklist de cutover)
- [[Tray - Autenticação]] (api_host por loja; chaves não mudam por ambiente)
- [[heziom-api — Referência Técnica]] (envs Netlify)

## Escopo técnico
- Centralizar por loja: `store_id`, `api_host`, `consumer_key/secret`, `seller_id`, `app_code`, `access_token`, `refresh_token`. Fonte única = tabela `tray_tokens`/config por `store_key` (não env solto).
- `api_host` SEMPRE lido da resposta do `POST /auth` e persistido; nunca URL fixa.
- Remover hardcodes na `heziom-api` (ex.: `event_source_url` = `https://www.editoraheziom.com.br/`, versão Graph API) → derivar de config.
- Edges (`crm-tray-*`, `hub-tray-*`) e functions Netlify resolvem a loja por config, não por constante.

## Critérios de Aceite
- [ ] CA1 — Nenhum `store_id`/`api_host`/`seller_id`/`app_code`/URL de loja hardcoded no código (grep limpo).
- [ ] CA2 — `api_host` usado em qualquer chamada vem da persistência por loja (origem = resposta do `/auth`).
- [ ] CA3 — Trocar a config para uma segunda "loja fictícia" faz todas as edges apontarem para ela sem recompilar/editar código (dry-run).
- [ ] CA4 — Runbook de cutover escrito e revisado: passo a passo (instalar app na 1345958 → code → seed token → trocar config → re-registrar webhook → smoke) com rollback.
- [ ] CA5 — Segredos a rotacionar no cutover listados (consumer_secret, service key, CAPI token, webhook secret).

## Prontidão para produção
Esta story É a prontidão de cutover. Entregável final: runbook versionado no vault/repo + config de produção pré-preenchida (sem segredos) aguardando o `code` da loja 1345958.

## Dependências
