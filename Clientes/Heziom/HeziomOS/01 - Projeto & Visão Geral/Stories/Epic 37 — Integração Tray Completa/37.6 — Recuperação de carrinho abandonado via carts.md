# 37.6 — Recuperação de carrinho abandonado via `GET /carts`

## Contexto
> O Epic 6.23 (régua de carrinho abandonado) está no ar mas **DORME** por 4 pré-condições, uma delas era o endpoint de carrinho. A auditoria corrigiu: **não existe `/abandoned-carts`** — a recuperação é feita sobre a **API de Listagem de Carrinho** (`GET /carts`, paginada) + `GET /carts/{session_id}/complete`. Como não há webhook de carrinho, é **polling**.

## Spec de Referência
- [[Tray - Carrinho Abandonado e Scripts]] (corrigido 08/07)
- [[Tray — Auditoria de Capacidades vs Produção]] (ressalva 1: carrinho abandonado)
- Epic 6.23 (régua dormente) e edge `crm-tray-poll-abandoned-carts`

## Escopo técnico
- Corrigir a edge `crm-tray-poll-abandoned-carts` para usar `GET /carts` (paginado) + `GET /carts/{session_id}/complete`; corrigir env `TRAY_ABANDONED_CARTS_PATH` → `/carts`.
- Detecção de "abandonado": carrinho não finalizado há > X min, com e-mail e total > limiar (regra do 6.23).
- Disparar régua (templates `carrinho_abandonado_1/2/3` na Meta) + atribuir receita recuperada.
- Ativar o flow: `SELECT crm.install_abandoned_cart_flow(<admin>)` + cron `*/10` + `status='active'`.

## Critérios de Aceite
- [ ] CA1 — `GET /carts` retorna carrinhos na loja de teste; paginação percorre todos.
- [ ] CA2 — Detecção de abandono aplica a regra (tempo, e-mail, limiar) corretamente.
- [ ] CA3 — Régua dispara (Email/WhatsApp) para um carrinho de teste abandonado.
- [ ] CA4 — Receita recuperada é atribuída quando o carrinho vira pedido.
- [ ] CA5 — Env `TRAY_ABANDONED_CARTS_PATH` corrigido; nenhuma referência a `/abandoned-carts`.
- [ ] CA6 — 4 pré-condições do 6.23 satisfeitas (templates Meta aprovados, Tray conectada, endpoint confirmado, cron+flow ativos).

## Prontidão para produção
Store-agnostic. Templates Meta e flow são operacionais (não-código) — deixar documentados no runbook. Validar o formato real de "carrinho não finalizado" na loja de teste (a doc trata abandono como caso de uso sobre a listagem, não recurso próprio).

## Dependências
- 37.3 (path correto), 37.4 (polling), Epic 6.23, templates Meta aprovados.
