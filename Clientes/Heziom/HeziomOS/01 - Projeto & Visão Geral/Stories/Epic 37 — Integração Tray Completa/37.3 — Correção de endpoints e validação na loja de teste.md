# 37.3 — Correção de endpoints e validação na loja de teste

## Contexto
> A auditoria achou 6 endpoints errados nos docs (e potencialmente no código, que foi escrito em cima deles). Antes de construir os fluxos das Fases 2/3, corrigir e **validar cada endpoint real na loja de teste 1501119**.

## Spec de Referência
- [[Tray — Auditoria de Capacidades vs Produção]] (tabela "Correções da API oficial")
- Notas de origem já corrigidas (08/07): [[Tray - Carrinho Abandonado e Scripts]], [[Tray - Cupons e Promoções]], [[Tray - Invoices]], [[Tray - Pedidos]], [[Tray - Clientes]]

## Escopo técnico — de/para a corrigir
| Errado | Correto |
|---|---|
| `GET /abandoned-carts` | `GET /carts` (paginado) + `GET /carts/{session_id}/complete` |
| `/coupons` | `/discount_coupons` (+ `create_relationship`) |
| `POST /invoices` | `POST /orders/:order_id/invoices` (retorna DANFE) |
| `PUT /orders/:id {sending_code}` | `PUT /orders/:id {status_id, tracking_number}` |
| "Tray não segmenta cliente" | `/customers/profiles` + `/price-lists` existem |

- Varrer edges (`crm-tray-*`, `hub-tray-*`) e `heziom-api` por esses paths.
- Corrigir env `TRAY_ABANDONED_CARTS_PATH` (hoje `/abandoned-carts`).
- Confirmar o padrão de base URL: `{api_host}/web_api/{recurso}` (algumas notas usam `/web_api/v2/`; validar qual responde).

## Critérios de Aceite
- [ ] CA1 — `GET /carts` retorna carrinhos na loja de teste (ou limitação de sandbox documentada).
- [ ] CA2 — `GET /discount_coupons` retorna cupons na loja de teste.
- [ ] CA3 — `POST /orders/:order_id/invoices` aceita um registro de NF de teste (ou 404 documentado como limitação `implantacao`).
- [ ] CA4 — `PUT /orders/:id` com `status_id`/`tracking_number` altera o pedido na loja de teste.
- [ ] CA5 — `GET /customers/profiles` e `GET /price-lists` respondem na loja de teste.
- [ ] CA6 — Grep no código: nenhum path antigo remanescente; env corrigido.
- [ ] CA7 — Registro de request/response (URLs + payloads) de cada endpoint — reaproveitável como evidência de homologação (37.4).

## Prontidão para produção
Endpoints são os mesmos em qualquer loja; validação em teste transfere para produção. Onde a loja de teste (em `implantacao`) bloquear (ex.: NF, hooks), marcar explicitamente como "reteste obrigatório em produção".

## Dependências
- Base para 37.5 (payments), 37.6 (carts), 37.7 (invoices), 37.8 (profiles/price-lists).
