## 6. O que o HeziomOS precisa replicar (por prioridade)

### Métricas de CRM obrigatórias no HeziomOS

| Métrica | Origem dos dados | Prioridade |
|---|---|---|
| **Taxa de Recompra** (global + por mês) | Tray orders + Literarius | 🔴 Alta |
| **Receita Total** por período | Tray + Literarius + marketplaces | 🔴 Alta |
| **Receita por tipo de cliente** (novo vs. recorrente) | Tray orders com flag de 1ª compra | 🔴 Alta |
| **Ticket Médio** por pedido e por cliente | Tray orders | 🔴 Alta |
| **Pedidos por cliente** | Tray orders agrupado por customer_id | 🔴 Alta |
| **Tempo médio de recompra** | Diferença entre datas de pedidos | 🟠 Média |
| **Clientes pagantes únicos** | Tray + Literarius | 🔴 Alta |
| **Captação diária** (novos leads) | Flowbiz/HeziomOS lead capture | 🟠 Média |
| **ROI de campanhas** (receita influenciada / custo) | Atribuição por UTM/email_id | 🟠 Média |
| **Taxa de Abertura / CTOR** por campanha | Provedor e-mail (Resend/SES) | 🟠 Média |
| **Funil de conversão** (envio → abertura → clique → pedido) | E-mail provider + Tray webhook | 🟠 Média |

### Insights estratégicos para o HeziomOS

1. **Recompra é o maior alavancador:** ticket 26% maior, margem melhor. Focar réguas de recompra em D+75 (tempo médio 74 dias).
2. **69% da receita vem de novos clientes** — dependência perigosa. HeziomOS deve ter alerta de "clientes inativos há X dias" com disparo automático.
3. **Taxa de recompra atual: 25,89%.** Benchmark e-commerce editorial BR: 30–40%. Gap de ~10pp = ~R$270.000/ano de receita adicional potencial.
4. **Captação funciona (119% da meta).** LPs e pop-ups já estão rodando. Focar em aumentar engajamento pós-captura (onboarding automático).
5. **WhatsApp tem taxa de abertura >74%** na única jornada ativa. Ativar campanhas WhatsApp seria multiplicar engajamento sem aumentar custo.
6. **Jun/26 com 34,91% de recompra** — o Plano Bomba parece estar funcionando para reativar clientes antigos.

## 7. Schema de dados necessário no HeziomOS (CRM Dashboard)

Para replicar o dashboard CRM do Flowbiz com dados próprios, o HeziomOS precisará dessas queries/views no Supabase:

```sql
-- Taxa de Recompra mensal
SELECT 
  DATE_TRUNC('month', order_date) as mes,
  COUNT(DISTINCT CASE WHEN order_number > 1 THEN customer_id END)::float 
    / COUNT(DISTINCT customer_id) * 100 as taxa_recompra,
  COUNT(DISTINCT CASE WHEN order_number = 1 THEN customer_id END) as novos_clientes,
  COUNT(DISTINCT CASE WHEN order_number > 1 THEN customer_id END) as clientes_recorrentes
FROM orders
GROUP BY 1 ORDER BY 1;

-- Ticket médio por tipo de cliente
SELECT
  CASE WHEN order_number = 1 THEN 'Novo' ELSE 'Recorrente' END as tipo,
  AVG(order_value) as ticket_medio,
  COUNT(*) as pedidos,
  SUM(order_value) as receita_total
FROM orders
GROUP BY 1;

-- Tempo médio de recompra (dias entre pedidos)
SELECT AVG(days_to_reorder) as tempo_medio_recompra
FROM (
  SELECT customer_id,
    EXTRACT(DAY FROM order_date - LAG(order_date) OVER (PARTITION BY customer_id ORDER BY order_date)) as days_to_reorder
  FROM orders
) t WHERE days_to_reorder IS NOT NULL;
```
